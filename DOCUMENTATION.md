## Overview

This toolkit replaces the NOAA-EMC **gridgen** MATLAB package with a pure-Python
workflow. Given only a bounding box and a grid resolution, it produces the four
ASCII grid files that WW3's pre-processor (`ww3_grid`) requires:

| File              | Contents                                                         |
| ----------------- | ---------------------------------------------------------------- |
| `{prefix}.dep`  | Bathymetry (integers, millimetres, scale factor 0.001 → metres) |
| `{prefix}.mask` | Land/sea mask (0 = land, 1 = ocean, 2 = open boundary)           |
| `{prefix}.obs`  | Sub-grid obstacles (percentage blockage per cell edge)           |
| `{prefix}.meta` | Grid metadata for embedding in `ww3_grid.inp`                  |

The output format is identical to that produced by MATLAB gridgen, so the files
can be used directly with any WW3 build.

```bash
pip install numpy scipy xarray netcdf4 pyyaml matplotlib
```

| Package        | Purpose                                                               |
| -------------- | --------------------------------------------------------------------- |
| `numpy`      | Array operations                                                      |
| `scipy`      | `RegularGridInterpolator` / `griddata` for bathymetry re-gridding |

| `netcdf4`    | NetCDF backend for xarray                                             |
| `pyyaml`     | Parsing `config_bounds.yaml`                                        |
| `matplotlib` | Plotting in `prepMaskWW3.py`                                        |

**Optional — download bathymetry on-the-fly:**

```bash
pip install pygmt        # also requires GMT installed on the system
```

Download GEBCO 2023 global grid (free) from [https://www.gebco.net/](https://www.gebco.net/).

## Quickstart — Generating a Grid

### Step 1 — Edit the configuration file

Open `scripts/config_bounds.yaml` and set your domain:

```yaml
grid:
  lon_min: -46.0
  lon_max: -38.0
  lat_min: -26.0
  lat_max: -18.0
  dx: 0.05          # grid spacing in degrees (~5.5 km at these latitudes)
  dy: 0.05
  zdep: -0.10        # depth threshold: cells shallower than this are LAND
  zmin:  2.50        # minimum water depth allowed in WW3

bathymetry:
  source: file
  file: /path/to/gebco_2023.nc   # any GEBCO / ETOPO NetCDF

output:
  directory: my_grid
  prefix:    mygrid
```

### Step 2 — Run the generator

```bash
cd scripts
python genWW3grid_fromBounds.py config_bounds.yaml
```

The script prints a summary and creates four files under `my_grid/`:

```
my_grid/
├── mygrid.dep
├── mygrid.mask
├── mygrid.obs
└── mygrid.meta
```

### Step 3 (optional) — Add open-boundary markers

If the grid will be **nested inside a larger WW3 grid**, the edges of the mask
must be set to `2` (open-boundary condition).  Run:

```bash
python prepMaskWW3.py my_grid/mygrid.mask
```

This overwrites ocean cells on the four border rows/columns with the value `2`
and saves `my_grid/mygrid.mask2`.

### Step 4 (optional) — Interpolate to a finer resolution

If you want to increase resolution (e.g., from 3 km to 2 km) without
regenerating from bathymetry data:

```bash
# First edit the hardcoded domain values in interpBatMaskWW3.py / interpObsWW3.py
# (see Known Limitations)
python interpBatMaskWW3.py
python interpObsWW3.py
```

### Step 5 — Use in ww3_grid.inp

Copy the content of `mygrid.meta` into the **grid definition section** of
`ww3_grid.inp`, or reference it with the `INPGRD` namelist option, adjusting
file paths as needed.

## Script Reference

### genWW3grid_fromBounds.py

**Purpose:** Main grid-generation script. Reads the YAML config, builds the
regular grid, interpolates bathymetry, derives the land/sea mask, and writes all
four WW3 ASCII files.

**Usage:**

```bash
python genWW3grid_fromBounds.py [config_file.yaml]
# default config file: config_bounds.yaml
```

**Config keys:**

| Key                             | Type  | Required            | Description                                    |
| ------------------------------- | ----- | ------------------- | ---------------------------------------------- |
| `grid.lon_min`                | float | ✓                  | West bound (degrees, −180–180 or 0–360)     |
| `grid.lon_max`                | float | ✓                  | East bound                                     |
| `grid.lat_min`                | float | ✓                  | South bound                                    |
| `grid.lat_max`                | float | ✓                  | North bound                                    |
| `grid.dx`                     | float | ✓                  | Longitude spacing (degrees)                    |
| `grid.dy`                     | float | ✓                  | Latitude spacing (degrees)                     |
| `grid.zdep`                   | float | —                  | Depth threshold m (default −0.10)             |
| `grid.zmin`                   | float | —                  | Min depth for WW3 m (default 2.50)             |
| `bathymetry.source`           | str   | —                  | `file` (default) or `pygmt`                |
| `bathymetry.file`             | str   | ✓ (if source=file) | Path to NetCDF bathymetry                      |
| `bathymetry.variable`         | str   | —                  | Variable name (auto-detected if omitted)       |
| `bathymetry.pygmt_resolution` | str   | —                  | e.g.`01m`, `30s` (default `01m`)         |
| `output.directory`            | str   | —                  | Output directory (default `ww3_grid_output`) |
| `output.prefix`               | str   | —                  | File prefix (default `grid`)                 |

**Internal workflow:**

```
load_config()
    ↓
Build regular lon/lat arrays  (np.arange)
    ↓
load_bathymetry()
  ├─ load_bathy_from_file()   ← NetCDF via xarray
  └─ load_bathy_pygmt()       ← on-the-fly download
    ↓
interpolate_to_ww3_grid()    ← scipy.RegularGridInterpolator (bilinear)
    ↓
make_mask()                  ← elevation < zdep  → ocean (1), else land (0)
    ↓
write_dep()  / write_mask()  / write_obs()  / write_meta()
```

**Grid size calculation:**

```
NX = floor((lon_max − lon_min) / dx) + 1
NY = floor((lat_max − lat_min) / dy) + 1
```

The actual eastern/northern edge is `lon_min + (NX−1)*dx`, which may be
slightly less than `lon_max` due to floating-point flooring — this matches the
convention used by MATLAB gridgen.

### prepMaskWW3.py

**Purpose:** Marks ocean cells on all four border rows/columns with value `2`
(WW3 open-boundary condition).  Equivalent to `maskmod.m` in the MATLAB
gridgen examples.

**Usage:**

```bash
python prepMaskWW3.py <mask_file>
# e.g.:
python prepMaskWW3.py my_grid/mygrid.mask
```

**Logic:**

- Reads mask with `np.loadtxt`.
- Iterates over the **second row from each edge** (indices 1 and NY−2 for
  rows; 1 and NX−2 for columns).  Cells that are already `0` (land) are
  skipped.
- Saves the result as `<input_file>2` (e.g., `mygrid.mask2`).
- Displays a `pcolormesh` plot for visual inspection.

> **Note:** The row/column indices are 1-based in the original MATLAB code
> (`i=2…NY−1`, `j=2…NX−1`) but translated to 0-based Python indices `1` and
> `N−2`. This preserves the outermost ring as land (WW3 requirement).

---

### interpBatMaskWW3.py

**Purpose:** Interpolates an existing low-resolution `.dep` and `.mask` to a
finer grid using nearest-neighbour interpolation (`scipy.interpolate.griddata`).

**Current limitations:** Domain bounds and grid sizes are **hardcoded** to the
Rio de Janeiro example (see [Known Limitations](#known-limitations-and-differences)).

**Usage** (after editing hardcoded values):

```bash
python interpBatMaskWW3.py
```

**Key parameters (edit in script):**

```python
latMin, latMax = -26.8, -19.5
lonMin, lonMax = 314.0, 322.0
nLon1, nLat1   = 223, 203    # original resolution
nLon2, nLat2   = 446, 406    # target resolution (2× in each direction)
```

---

### interpObsWW3.py

**Purpose:** Interpolates an existing `.obs` file to a finer resolution.

**Important:** The `.obs` file has `2*NY` rows (NY rows for E–W face
obstructions + NY rows for N–S face obstructions).  This script accounts for
that by using `nLat = 2 * nLat1`.

**Current limitations:** Same hardcoded domain as `interpBatMaskWW3.py`.

## Output File Formats

All formats match those produced by MATLAB gridgen and expected by `ww3_grid`.

### `.dep` — Bathymetry

- One row per latitude (south → north, `IDLA=1`).
- Each row contains `NX` integer values, **12 characters wide each**.
- Values in **millimetres** (multiply by `0.001` to get metres).
- Land sentinel: `999999000` mm (= 999,999 m).
- Negative values = ocean depth; positive = above MSL (also treated as depth).

```
  -15340    -8920   999999000   -4210  ...
```

### `.mask` — Land/Sea Mask

- One row per latitude (south → north).
- Each row contains `NX` integer values, **3 characters wide each**.
- `0` = land, `1` = ocean, `2` = open boundary.

```
  0  1  1  1  0 ...
```

### `.obs` — Sub-grid Obstacles

- `2*NY` rows × `NX` columns, **3 characters wide each**.
- Rows `0…NY−1` : E–W face obstruction (%).
- Rows `NY…2NY−1`: N–S face obstruction (%).
- This script always writes all zeros (no real obstacle data computed).

### `.meta` — Grid Metadata

Plain-text file with comment lines (starting with `$`) and WW3 grid parameters.
Designed to be pasted directly into `ww3_grid.inp`.

Example (Rio de Janeiro, 2.16′ resolution):

```
   'RECT'  T  'NONE'
    223       203
   2.16      2.16     60.00
314.0000         -26.8000         1.00
$ Bottom Bathymetry
-0.10   2.50  40  0.001000  1  1 '(....)'  NAME  './rio.dep'
$ Sub-grid information
50  0.010000  1  1  '(....)'  NAME  './rio.obs'
$ Mask Information
60  1  1  '(....)'  NAME  './rio.mask'
```

The grid increments are stored in arc-minutes (`SX = dx * 60`) with a scale
factor of 60, matching the convention used in MATLAB gridgen.

## Comparison with the Original MATLAB gridgen

The original [NOAA-EMC/gridgen](https://github.com/NOAA-EMC/gridgen) package is
a comprehensive MATLAB toolbox (last updated 2021) distributed with WW3. It was
developed at NOAA/NCEP and supports both rectilinear and curvilinear grids.

### MATLAB gridgen — Workflow Summary

```
create_grid.m (master script)
  │
  ├─ generate_grid.m        ← reads ETOPO1 or ETOPO2 from NetCDF;
  │                            computes cell-corner geometry;
  │                            bi-linear interpolation (fine grids) OR
  │                            area-weighted cell averaging (coarse grids)
  │
  ├─ compute_boundary.m     ← extracts GSHHS coastal polygons for domain
  ├─ split_boundary.m       ← splits large polygons for efficiency
  │
  ├─ clean_mask.m           ← refines land/sea mask using inpolygon()
  │                            against GSHHS coastline database
  │
  ├─ remove_lake.m          ← identifies and removes disconnected water
  │                            bodies (lakes, isolated embayments)
  │
  ├─ create_obstr.m         ← computes % blockage at each cell edge using
  │                            coastal polygon–cell edge intersections
  │
  ├─ write_ww3file.m        ← writes .dep and .mask (integer ASCII)
  ├─ write_ww3obstr.m       ← writes .obs (2*NY rows × NX cols)
  └─ write_ww3meta.m        ← writes .meta (supports RECT and CURV)
```

**Reference data required by MATLAB gridgen:**

- `etopo1.nc` or `etopo2.nc` (global bathymetry, ~500 MB)
- `coastal_bound_{coarse|low|inter|high|full}.mat` (GSHHS polygon databases)
- Optional user-defined polygon `.flag` file

### Side-by-Side Feature Comparison

| Feature                                | MATLAB gridgen                             | Python myWW3tools                             |
| -------------------------------------- | ------------------------------------------ | --------------------------------------------- |
| **Grid geometry**                | Rectilinear**and** curvilinear       | Rectilinear only                              |
| **Bathymetry source**            | ETOPO1 / ETOPO2 (required)                 | Any NetCDF (GEBCO, ETOPO, etc.) or pygmt      |
| **Bathymetry interpolation**     | Bilinear (fine) / area-average (coarse)    | Bilinear (`RegularGridInterpolator`)        |
| **Mask generation**              | Depth threshold + GSHHS polygon refinement | Depth threshold only                          |
| **Lake / isolated body removal** | Yes (`remove_lake.m`)                    | **No**                                  |
| **Sub-grid obstructions**        | Real (polygon × cell-edge intersection)   | **All zeros**                           |
| **Open-boundary markers**        | `maskmod.m` (manual, post-step)          | `prepMaskWW3.py` (same logic)               |
| **Resolution re-gridding**       | Not included                               | `interpBatMaskWW3.py` / `interpObsWW3.py` |
| **Configuration**                | Hardcoded in `.m` script                 | YAML config file                              |
| **Coordinate convention**        | 0–360° only (internal)                   | −180–180° or 0–360° (auto-detected)      |
| **Output format**                | Identical WW3 ASCII format                 | Identical WW3 ASCII format                    |
| **Dependency**                   | MATLAB ≥ r2008a + NetCDF toolbox          | Python 3 + numpy/scipy/xarray                 |
| **Global grid support**          | Yes (`IS_GLOBAL=1`, SMPL closure)        | Yes (auto-detected)                           |
| **Curvilinear grids**            | Yes                                        | No                                            |
| **Visualization**                | MATLAB `pcolor` figures                  | `prepMaskWW3.py` matplotlib plot            |

### Interpolation Method Detail

**MATLAB `generate_grid.m`** uses an adaptive strategy:

- If the target cell is **finer** than the source grid (`ndx ≤ 1 && ndy ≤ 1`):
  four-point bilinear interpolation using the four surrounding source cells.
- If the target cell is **coarser** than the source grid: all source points
  inside the cell polygon are found with `inpolygon`, and their average is used
  if the wet fraction exceeds the `CUT_OFF` threshold; otherwise the cell is
  set to `DRY_VAL`.

**Python `genWW3grid_fromBounds.py`** always uses scipy's
`RegularGridInterpolator` with `method='linear'` (bilinear), regardless of
relative resolution. For grids that are coarser than the source, this is
equivalent to point-sampling rather than area-averaging — results will be
acceptable for most applications but may differ slightly from MATLAB in highly
heterogeneous coastal zones.

### Mask Refinement Detail

**MATLAB gridgen** performs a two-step mask refinement:

1. Initial dry/wet flag from `generate_grid` (depth threshold).
2. `clean_mask.m` uses the **GSHHS coastal polygon database** (5 resolution
   levels from 25 km coarse to full resolution) to correct coastal cells via
   `inpolygon` checks, ensuring that the mask correctly follows the actual
   shoreline even when source bathymetry is ambiguous near the coast.
3. `remove_lake.m` labels disconnected water regions and removes them from the
   ocean mask.

**Python `make_mask()`** applies a single depth threshold:
`mask = 1` where `elevation < zdep` (default −0.10 m), `0` elsewhere.
No polygon refinement is performed. In practice this is adequate when using
high-resolution bathymetry (GEBCO 15-arc-second), but may misclassify some
shallow coastal cells or leave isolated lake pixels classified as ocean.

## Known Limitations and Differences

1. **Hardcoded values in `interpBatMaskWW3.py` and `interpObsWW3.py`**The domain bounds and grid sizes are set to the Rio de Janeiro example.
   Before using these scripts for a different domain, edit the seven parameters
   at the top of each file.
2. **No real sub-grid obstructions** `genWW3grid_fromBounds.py` writes an all-zero `.obs` file.Sub-grid obstructions matter mainly for wave propagation through narrow
   straits (channels narrower than ~3× the grid cell). If your domain contains
   significant narrow passages, consider post-processing with a dedicated tool
   or the original MATLAB `create_obstr.m`.
3. **No lake removal**Isolated water bodies (lakes, inland seas) will appear as ocean points in the
   mask unless their elevation is positive. Check the mask visually with
   `prepMaskWW3.py` or a GIS tool.
4. **Coordinate convention**MATLAB gridgen enforces 0–360° longitude internally. The Python scripts
   accept both −180–180° and 0–360°, but mixing conventions within a single
   run (e.g., `lon_min = -46`, source bathymetry in 0–360) is handled
   automatically by `interpolate_to_ww3_grid()`.
5. **No curvilinear grid support**The Python scripts only handle regular (rectilinear) grids. For polar,
   rotated, or stretched grids, the MATLAB gridgen is still required.
6. **`prepMaskWW3.py` boundary logic**
   The script marks only the **second** row/column from each edge (Python
   indices 1 and N−2), leaving the outermost ring as land — as required by WW3.
   For complex nested configurations, additional boundary adjustment may be
   needed manually.
